langrev. h1.View p.Custom View are blank areas within the window that allow us to implement our own components. They are characterized by three things: li.They are integrated within a Layout like the rest of interface elements. The layout will manage the position and size of the view based on its configuration. li.They allow you to draw anything through the OnDraw event. The handler will receive a 2D context ready to draw. li.They allow to capture the keyboard and mouse events produced by the user, which will make it possible to interact with the generated scenes. li.They assume nothing about the content. It is the programmer who must synchronize the user's events with the drawing to produce the desired effect. li.Use view_create to create a custom view. li.Use view_OnDraw to draw in a view. li.Use view_data to associate user data with the view. p.We can't draw in the view whenever we want. Being an area within the window is subject to overlaps so the drawing buffer is managed by the operating system. The refresh cycle is divided into several steps view_events: li.There is an event that requires updating the view content. li.We call the view_update method to notify that the view must be updated. li.In a timely manner, the system will send an OnDraw event with an DCtx object ready to receive our drawing command.
