langrev. h1.Button p.The buttons are another classic element in graphic interfaces, where we distinguish four types: the push button, checkbox, radiobutton and flat button typical of toolbars buttons . In Hello Button! you have an example of use. li.Use button_pushto create a push button. li.Use button_checkto create a check box. li.Use button_check3 to create a box with three states. li.Use button_radio to create a radio button. li.Use button_flatto create a flat button. li.Use button_flatgleto create a flat button with status. img(buttons.png,0,.45).Windows, macOS and Linux buttons. Window where several typical buttons appear: Push, check box and radio buttons. p.In addition to capturing the event and notifying the application, the checkbox and flatgle maintain a state (pressed/check or released/uncheck). li.Use button_stateto set the button status. li.Use button_get_state to get the status of the button. p.Special mention is required of the radio buttons, which only make sense when they appear in a group, since they are used to select a single option within a set. Groups are formed at the Layout level, that is, all radiobuttons of the same layout will be considered from the same group, where only one of them can be selected. If we need several sub-groups, we must create several sub-layout, as shown radiogroup laygroup. When capturing the event, the field indexfrom EvButton will indicate the index of the button that has been pressed. img(radiogroup.svg,480,.65).Radio groups linked to different layouts.
