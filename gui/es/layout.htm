h1.Layout p.Un Layout es una rejilla virtual y transparente vinculada siempre con un Panel que sirve para ubicar los diferentes elementos de la interfaz simple_layout. Sus celdas interiores tienen la capacidad de dimensionarse automáticamente en función de su contenido, con lo que se consigue una gran portabilidad debido a que no es necesario indicar coordenadas ni tamaños específicos para los controles. En dimnat tienes las medidas por defecto de cada control. El ancho de columna se fija al del elemento más ancho y de igual forma se calcula la altura de las filas. No es difícil darse cuenta que esta disposición no encaja de ninguna manera en una rejilla rectangular, por lo que ha llegado el momento de utilizar sublayouts. Además de controles individuales, una celda admite también otro layout, por lo que podemos dividir el panel original en tantas partes como sean necesarias hasta conseguir la maquetación deseada. El layout principal irá dimensionando cada sublayout de forma recursiva e integrándolo en la composición final. En ¡Hola Sublayout! tienes el código que genera este ejemplo. li.Utiliza layout_layout para asignar un layout completo a una celda de otro layout. img(complex_layout.png,0,.75).Composición de panel compleja. Esto ocurre cuando tenemos un sublayout en una celda con alineación ekJUSTIFY (expansión interna) o cuando el usuario cambia el tamaño de una ventana re-dimensionable (expansión externa). Esto producirá un "exceso de píxeles" entre el dimensionado natural y el tamaño real de la celda layout_expand. Esta situación se resuelve repartiendo el sobrante por igual entre todas las columnas del sublayout, que a su vez, se irán expandiendo de forma recursiva hasta llegar a una celda vacía o un control individual. Podemos cambiar este reparto equitativo mediante estas funciones: li.Utiliza layout_hexpand para expandir una única celda y dejar el resto con su tamaño por defecto. li.Utiliza layout_hexpand2 para expandir dos celdas indicando la proporción de crecimiento de cada una. li.Utiliza layout_hexpand3 para expandir tres celdas. img(layout_expand.svg,520,.65).Cuando el tamaño del sublayout viene dado por condiciones externas, se reparte por igual el exceso de píxeles entre las columnas (expansión horizontal) y filas (expansión vertial). Comportamiento de un sublayout cuando su tamaño se expande para ajustarse a una celda. bq.La expansión vertical funciona exactamente igual, repartiendo el espacio sobrante entre las filas del layout.
