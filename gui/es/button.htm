h1.Button p.Los botones son otro elemento clásico en interfaces gráficas, donde distinguimos cuatro tipos: El botón de pulsación, casilla de verificación (checkbox), botón de radio radiobutton y botón plano típico de las barras de herramientas buttons. En ¡Hola Button! tienes un ejemplo de uso. li.Utiliza button_push para crear un botón de pulsación. li.Utiliza button_check para crear una casilla de verificación. li.Utiliza button_check3 para crear una casilla con tres estados. li.Utiliza button_radio para crear un botón de radio. li.Utiliza button_flat para crear un botón plano. li.Utiliza button_flatgle para crear un botón plano con estado. img(buttons.png,0,.45).Botones en Windows, macOS y Linux. Ventana donde aparecen varios botones típicos: Pulsación, casilla de verificación y botones de radio. p.Además de capturar el evento y notificar a la aplicación, los checkbox y flatgle mantienen un estado (pulsado/check o liberado/uncheck). li.Utiliza button_state para establecer el estado del botón. li.Utiliza button_get_state para obtener el estado del botón. p.Mención especial requieren los botones de radio, que solo tienen sentido cuando aparecen en grupo ya que se utilizan para seleccionar una única opción dentro de un conjunto. Los grupos se forman a nivel de Layout, es decir, todos los radiobutton de un mismo layout se considerarán del mismo grupo, donde solo uno de ellos podrá estar seleccionado. Si necesitamos varios sub-grupos, deberemos crear varios sub-layout, como se muestra en radiogroup laygroup. A la hora de capturar el evento, el campo index de EvButton indicará el índice del botón que ha sido pulsado. img(radiogroup.svg,480,.65).Radio grupos vinculados a diferentes layouts. Ejemplos de radiogrupos. Un grupo de ocho opciones y dos grupos de cuatro. code(cpp,laygroup).Grupos de botones de radio.
