h1.View p.Las View o vistas personalizadas son áreas en blanco dentro de la ventana que nos permiten implementar nuestros propios componentes. Se caracterizan por tres cosas: li.Están integradas dentro de un Layout como el resto de elementos de interfaz. El layout gestionará la posición y tamaño de la vista en función de su configuración. li.Permiten dibujar cualquier cosa a través del evento OnDraw. El manejador recibirá un contexto 2D listo para dibujar. li.Permiten capturar los eventos de teclado y ratón producidos por el usuario, lo que hará posible interactuar con las escenas generadas. li.No asumen nada sobre el contenido. Es el programador el que debe sincronizar los eventos del usuario con el dibujo para producir el efecto deseado. li.Utiliza view_create para crear una vista personalizada. li.Utiliza view_OnDraw para dibujar en una vista. li.Utiliza view_data para asociar datos de usuario con la vista. p.No podemos dibujar en la vista cuando queramos. Al ser un área dentro de la ventana está sujeta a solapamientos por lo que el búfer de dibujo es gestionado por el sistema operativo. El ciclo de refresco se divide en varios pasos view_events: li.Se produce algún evento que requiere actualizar el contenido de la vista. li.Llamamos al método view_update para notificar que la vista debe ser actualizada. li.En el momento oportuno, el sistema mandará un evento OnDraw con un objeto DCtx preparado para recibir nuestros comando de dibujo.
