h1.Primitivas de dibujo p.En el capítulo anterior realizamos una introducción al dibujo vectorial y vimos como trabajar con contextos 2D. Ahora veremos como dibujar líneas, figuras, texto e imágenes. En drawhello tienes la aplicación que integra todo lo que iremos viendo a continuación. h2.Dibujo de líneas p.La operación más elemental es dibujar una recta entre dos puntos. En contextos 2d las líneas son objetos sólidos y no una mera hilera de píxeles. Pensemos que estamos utilizando rotuladores de punta gruesa, donde la línea teórica permanecerá siempre en el centro del trazo line_draw. Podemos cambiar la forma de las terminaciones (linecap), de las uniones (linejoin) y establecer un patrón para líneas discontinuas. li.draw_line dibujará una línea. li.draw_polyline dibujará varias líneas conectadas. li.draw_line_color establecerá el color de línea. li.draw_line_width establecerá el ancho de línea. li.draw_line_cap establecerá el estilo de los extremos. li.draw_line_join establecerá el estilo de las uniones. li.draw_line_dash establecerá un patrón de puntos. img(line_draw.png,0,.65).Diferentes estilos línea. (a) ekLCFLAT. (b) ekLCSQUARE. (c) ekLCROUND. (d) ekLJMITER. (e) ekLJROUND. (f) ekLJBEVEL. Los patrones: [5, 5, 10, 5], [1, 1], [2, 1], [1, 2], [5, 5, 10, 5], NULL. Imagen con diferentes tipos de línea, grosos, color, etc. h2.Figuras y contornos p.Para dibujar figuras o áreas cerradas tenemos los siguientes comandos: li.draw_rect para rectángulos. li.draw_rndrect para rectángulos con bordes redondeados. li.draw_circle para círculos. li.draw_ellipse para elipses. li.draw_polygon para polígonos. li.draw_fill_color establece el color de relleno de regiones. p.Como vemos en shapes_draw podemos dibujar el contorno de la figura, su interior o ambos. Para el contorno se tendrá en cuenta el estilo de línea establecido según hemos visto en el apartado anterior. img(shapes_draw.png,0,.65).Solo contorno ekSTROKE. Solo relleno ekFILL. Primero contorno, después relleno ekSKFILL. Primero relleno, después contorno ekFILLSK. Imagen con diferentes figuras geométricas: Contornos y rellenos. p.Como ya vimos en dctx, el orden en que se efectúan las operaciones importa. No es lo mismo rellenar y luego dibujar el contorno que viceversa. El centro del trazo coincidirá con el contorno teórico de la figura. h2.Gradientes p.Los gradientes permiten rellenar regiones utilizando un degradado en lugar de un color sólido gradient. Se definen varios colores base y su posición relativa a lo largo de un vector graddef. Las posiciones [0, 1] corresponden a los extremos y los valores dentro de este rango a las posibles paradas intermedias. Cada línea perpendicular al vector delimita un color uniforme que se extenderá indefinidamente hasta alcanzar los límites de la figura a rellenar. li.Utiliza draw_fill_linear para activar el relleno con degradados. li.Utiliza draw_fill_color para volver al relleno de color sólido. img(gradient.svg,650,.65).Gradientes lineales. El color se interpola a lo largo de un vector. Ejemplo de un gradientes lineales en varias direcciones. code(cpp,graddef).Definición de los gradientes de gradient. // (a) Gradient color_t color[4]; real32_t stop[4] = {0, .35f, .7f, 1}; color[0] = color_rgb(255, 0, 0); color[1] = color_rgb(0, 255, 0); color[2] = color_rgb(0, 0, 255); color[3] = color_rgb(255, 0, 255); draw_fill_linear(ctx, color, stop, 4, 0, 0, 600, 400); // (b) Gradient color_t color[2]; real32_t stop[2] = {0, 1}; color[0] = color_rgb(255, 0, 0); color[1] = color_rgb(0, 0, 255); draw_fill_linear(ctx, color, stop, 2, 0, 0, 600, 0); // (c) Gradient color_t color[2]; real32_t stop[2] = {0, 1}; color[0] = color_rgb(255, 0, 0); color[1] = color_rgb(0, 0, 255); draw_fill_linear(ctx, color, stop, 2, 0, 0, 0, 400); // (d) Gradient color_t color[2]; real32_t stop[2] = {0, 1}; color[0] = color_rgb(255, 0, 0); color[1] = color_rgb(0, 0, 255); draw_fill_linear(ctx, color, stop, 2, 0, 0, 600, 400); code. h3.Transformación del gradiente p.Dado que el gradiente se define mediante un vector, es posible establecer una transformación que cambie la forma en la que es aplicado. Esta matriz es totalmente independiente a la que se aplica a las primitivas de dibujo draw_matrix, como vimos en Sistemas de referencia. li.Utiliza draw_fill_matrix para establecer la transformación del gradiente. Con esto podemos conseguir varios efectos: li.Gradiente global: El degradado se aplicará de forma global al fondo, y las figuras serán recortes del mismo patrón gradient_global. Para ello estableceremos la matriz identidad como transformación del gradiente gradglob. Es la definida por defecto. img(gradient_global.gif,600,.65).Gradiente global. No se pierde la continuidad entre figuras. Animación de un gradiente lineal variando la dirección. code(cpp,gradglob).Matriz del gradiente para todo el dibujo. draw_fill_linear(ctx, c, stop, 2, 0, 0, 600, 400); draw_fill_matrix(ctx, kT2D_IDENTITYf); i_draw_shapes(ctx); code. li.Gradiente local: El vector es trasladado al origen de la figura o a un punto de su entorno cercano gradient_local. Con esto, conseguiremos aplicar el degradado de forma local y que solo afecte a una figura concreta. En gradlocal hemos variado ligeramente la transformación para fijar el origen en una esquina y no en el centro de la elipse. Esto puede variar en función del efecto deseado. img(gradient_local.gif,600,.65).Gradiente local. El origen se sitúa en la figura. Animación de varias figuras rellenas con un gradiente lineal variando la dirección. code(cpp,gradlocal).Matriz del gradiente para una figura. t2d_movef(&matrix, kT2D_IDENTITYf, 250, 280); t2d_rotatef(&matrix, &matrix, - kBMATH_PIf / 10); draw_matrix(ctx, &matrix); draw_fill_linear(ctx, c, stop, 2, 0, 0, 200, 100); t2d_movef(&matrix, &matrix, -100, -50); draw_fill_matrix(ctx, &matrix); draw_ellipse(ctx, ekSKFILL, 0, 0, 100, 50); code. h3.Límites del gradiente p.Como hemos comentado, el relleno de color se extenderá de manera uniforme e indefinida por todas las líneas perpendiculares al vector, pero... ¿Que ocurre fuera de sus límites? En wrapgrad gradient_wrap el gradiente se ha definido en x=[200, 400], siendo esta medida inferior a la figura a rellenar: li.Utiliza draw_fill_wrap para definir el comportamiento del gradiente fuera de límites. li.ekFCLAMP se utiliza el valor del extremo como constante en el área exterior. li.ekFTILE se repite el patrón de color. li.ekFFLIP se repite el patrón, pero invirtiendo el sentido lo que evita la pérdida de continuidad en el color. code(cpp,wrapgrad).Color uniforme fuera de los límites del gradiente gradient_wrap (a). draw_fill_linear(ctx, c, stop, 2, 200, 0, 400, 0); draw_fill_wrap(ctx, ekFCLAMP); draw_rect(ctx, ekFILLSK, 50, 25, 500, 100); code. img(gradient_wrap.svg,550,.65).Comportamiento en los límites: (a) ekFCLAMP, (b) ekFTILE, (c) ekFFLIP. Gradientes con diferentes tratamientos en el área exterior. h2.Dibujar texto p.El renderizado del texto es la parte más importante de la interfaz de usuario y una de las más complejas de resolver técnicamente. Antiguamente, se utilizaban pequeños bitmaps con la imagen de cada carácter, pero hoy en día el juego de carácteres unicode tiene miles de entradas que hay que dibujar con diferentes estilos. Los textos se pueden escalar, rotar, rellenar con diferentes patrones o maquetar en párrafos. Afortunadamente, toda esta casuística está resuelta por las APIs nativas de cada sistema operativo, lo que nos permite proporcionar una interfaz simplificada e intuitiva. li.Utiliza draw_text para dibujar textos en dctx. li.Utiliza draw_font para establecer la fuente tipográfica. li.Utiliza draw_text_width para establecer el ancho máximo de un bloque de texto. li.Utiliza draw_text_align para establecer la alineación del texto, con respecto a su punto de inserción. li.Utiliza draw_text_halign para establecer la alineación interna del texto. li.Utiliza draw_text_extents para obtener el tamaño de un bloque de texto. p.Para dibujar textos de una sola línea, tan solo debemos llamar a la función pertinente, pasando una cadena UTF8 simtext text_single_line. Previamente, podemos asignar la fuente, el color y su alineación. code(cpp,simtext).Dibujo de una línea de texto. Font *font = font_system(20, 0); draw_font(ctx, font); draw_fill_color(ctx, kCOLOR_BLUE); draw_text_align(ctx, ekLEFT, ekTOP); draw_text(ctx, ekFILL, "Text 文本 Κείμενο", 25, 25); code. img(text_single_line.png,0,.65).Textos de una sola línea, con alineación y transformaciones. Dibujo de una cadena de texto varias veces. p.Si la cadena a visualizar tiene saltos de línea (carácter '\n') se tendrán en cuenta y el texto se mostrará en varias líneas multext text_multiline. También podemos obtener la medida en píxeles de un bloque, útil para integrar el texto con otras primitivas. code(cpp,multext).Dibujo de textos con saltos de línea. const char_t *text = "Text new line\n文字换行\nΓραμμή κειμένου"; real32_t width, height; draw_text(ctx, ekFILL, text, 25, 25); draw_text_extents(ctx, text, -1, &width, &height); code. img(text_multiline.png,0,.65).Textos con carácter '\n'. Dibujo varias veces de una cadena de texto que contiene caracteres nueva línea. p.Si el texto no contiene saltos de línea, se dibujará de forma contínua expandiéndose horizontalmente. Esto puede no ser lo más apropiado en párrafos largos, por lo que podemos establecer un ancho máximo, forzando su dibujo en varias líneas blocktext text_paragraph. code(cpp,blocktext).Anchura máxima y alineación interna en bloques de texto. const char_t *text = "Lorem ipsum dolor sit amet...consequat"; draw_text_width(ctx, 200); draw_text_halign(ctx, ekLEFT); draw_text(ctx, ekFILL, text, 25, 25); draw_text_extents(ctx, text, 200, &width1, &height1); code. img(text_paragraph.png,0,.65).Párrafos de texto con límite de anchura. Así debe configurarse en el caso que tengamos que hacer un uso intensivo: Navegadores, procesadores de texto, informes, etc. Dibujar contornos o utilizar gradientes tendrá una penalización en el rendimiento. h2.Dibujar imágenes p.Imágenes generadas de forma procedimental o leídas desde disco pueden utilizarse como una primitiva de dibujo más dimage draw_image.
