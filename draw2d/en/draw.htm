langrev. h1.Drawing primitives p.In the previous chapter we made an introduction to vector drawing and saw how to work with 2D contexts. Now we will see how to draw lines, figures, text and images. In drawhello you have the application that integrates everything we will see next. h2.Line drawing p.The most elementary operation is to draw a line between two points. In 2d contexts the lines are solid objects and not a mere row of pixels. Let's think we are using thick tip pens, where the theoretical line will always remain in the center of the stroke line_draw. We can change the shape of the endings (linecap), the joints (linejoin) and establish a pattern for dashed lines. li.draw_line will draw a line. li.draw_polyline will draw several connected lines. li.draw_line_color will set the line color. li.draw_line_width set the line width. li.draw_line_cap set the style of the ends. li.draw_line_join set the style of the unions. li.draw_line_dash set a dot pattern. img(line_draw.png,0,.65).Different line styles. (a) ekLCFLAT. (b) ekLCSQUARE. (c) ekLCROUND. (d) ekLJMITER. (e) ekLJROUND. (f) ekLJBEVEL. The pattern: [5, 5, 10, 5], [1, 1], [2, 1], [1, 2], [5, 5, 10, 5], NULL. Image with different types of line, thicknesses, color, etc. h2.Figures and borders p.To draw closed figures or areas we have the following commands: li.draw_rect for rectangles. li.draw_rndrect for rectangles with rounded edges. li.draw_circle for circles. li.draw_ellipse for ellipses. li.draw_polygon for polygons. li.draw_fill_color set the area fill color. p.As we see in shapes_draw we can draw the outline of the figure, its interior or both. For the outline the line style established as we have seen in the previous section will be taken into account. img(shapes_draw.png,0,.65).Stroke only ekSTROKE. Fill only ekFILL. First stroke, then fill ekSKFILL. First fill, then stroke ekFILLSK. Image with different geometric figures: Stroke and fill. p.As we saw in dctx, the order in which the operations are carried out matters. It is not the same to fill in and then draw the outline as vice versa. The center of the stroke will coincide with the theoretical outline of the figure. h2.Gradients p.Gradients allow regions to be filled using a gradient instead of a solid color gradient. Several base colors and their relative position along a vector are defined graddef. The positions [0, 1] correspond to the extremes and the values within this range to the possible intermediate stops. Each line perpendicular to the vector defines a uniform color that will extend indefinitely until reaching the limits of the figure to be filled. li.Use draw_fill_linear to activate the fill with gradients. li.Use draw_fill_color to return to solid color fill. img(gradient.svg,650,.65).Linear gradients. The color is interpolated along a vector. Example of a linear gradients in several directions. code(cpp,graddef).Definition of gradient gradients. // (a) Gradient color_t color[4]; real32_t stop[4] = {0, .35f, .7f, 1}; color[0] = color_rgb(255, 0, 0); color[1] = color_rgb(0, 255, 0); color[2] = color_rgb(0, 0, 255); color[3] = color_rgb(255, 0, 255); draw_fill_linear(ctx, color, stop, 4, 0, 0, 600, 400); // (b) Gradient color_t color[2]; real32_t stop[2] = {0, 1}; color[0] = color_rgb(255, 0, 0); color[1] = color_rgb(0, 0, 255); draw_fill_linear(ctx, color, stop, 2, 0, 0, 600, 0); // (c) Gradient color_t color[2]; real32_t stop[2] = {0, 1}; color[0] = color_rgb(255, 0, 0); color[1] = color_rgb(0, 0, 255); draw_fill_linear(ctx, color, stop, 2, 0, 0, 0, 400); // (d) Gradient color_t color[2]; real32_t stop[2] = {0, 1}; color[0] = color_rgb(255, 0, 0); color[1] = color_rgb(0, 0, 255); draw_fill_linear(ctx, color, stop, 2, 0, 0, 600, 400); code. h3.Gradient transformation p.Since the gradient is defined by a vector, it is possible to set a transformation that changes the way it is applied. This matrix is totally independent from the one applied to drawing primitives draw_matrix, as we saw in Reference systems. li.Use draw_fill_matrix to set the gradient transformation. With this we can get several effects: li.Global gradient: The gradient will be applied globally to the background, and the figures will be cutouts of the same pattern gradient_global. To do this we will set the identity matrix as a gradient transformation gradglob. It is defined by default. img(gradient_global.gif,600,.65).Global gradient. The continuity between figures is not lost. Animation of a linear gradient varying the direction. code(cpp,gradglob).Gradient matrix for the whole drawing. draw_fill_linear(ctx, c, stop, 2, 0, 0, 600, 400); draw_fill_matrix(ctx, kT2D_IDENTITYf); i_draw_shapes(ctx); code. li.Local gradient: The vector is transferred to the origin of the figure or to a point in its near surroundings gradient_local. With this, we will be able to apply the gradient locally and that only affects a specific figure. In gradlocal we have slightly varied the transformation to fix the origin in a corner and not in the center of the ellipse. This may vary depending on the desired effect. img(gradient_local.gif,600,.65).Local gradient. The origin is placed in the figure. Animation of several figures filled with a linear gradient varying the direction. code(cpp,gradlocal).Gradient matrix for a figure. t2d_movef(&matrix, kT2D_IDENTITYf, 250, 280); t2d_rotatef(&matrix, &matrix, - kBMATH_PIf / 10); draw_matrix(ctx, &matrix); draw_fill_linear(ctx, c, stop, 2, 0, 0, 200, 100); t2d_movef(&matrix, &matrix, -100, -50); draw_fill_matrix(ctx, &matrix); draw_ellipse(ctx, ekSKFILL, 0, 0, 100, 50); code. h3.Gradient Limits p.As we have said, the color fill will spread evenly and indefinitely along all the lines perpendicular to the vector, but... What happens outside its limits? In wrapgrad gradient_wrap the gradient has been defined in x=[200, 400], this measure being lower than the figure to be filled: li.Use draw_fill_wrap to define the behavior of the gradient out of bounds. li.ekFCLAMP the end value is used as a constant in the outer area. li.ekFTILE the color pattern is repeated. li.ekFFLIP the pattern is repeated, but reversing the order which prevents the loss of continuity in color. code(cpp,wrapgrad).Uniform color outside the limits of the gradient gradient_wrap (a). draw_fill_linear(ctx, c, stop, 2, 200, 0, 400, 0); draw_fill_wrap(ctx, ekFCLAMP); draw_rect(ctx, ekFILLSK, 50, 25, 500, 100); code. img(gradient_wrap.svg,550,.65).Limit Behavior: (a) ekFCLAMP, (b) ekFTILE, (c) ekFFLIP. Gradients with different treatments in the outside area. h2.Drawing text p.Text rendering is the most important part of the user interface and one of the most complex to solve technically. In the past, small bitmaps were used with the image of each character, but nowadays the unicode character set has thousands of entries to draw with different styles. Texts can be scaled, rotated, filled in with different patterns or formatted in paragraphs. Fortunately, all this casuistry is solved by the native APIs of each operating system, which allows us to provide a simplified and intuitive interface. li.Use draw_text to draw texts in dctx. li.Use draw_font to set the font. li.Use draw_text_width to set the maximum width of a block of text. li.Use draw_text_align to set the alignment of the text, with respect to its insertion point. li.Use draw_text_halign to set the internal alignment of the text. li.Use draw_text_extents to get the size of a block of text. p.To draw single-line texts, we just have to call the function, passing a UTF8 string simtext text_single_line. Previously, we can assign the font, color and alignment. code(cpp,simtext).Dibujo de una línea de texto. Font *font = font_system(20, 0); draw_font(ctx, font); draw_fill_color(ctx, kCOLOR_BLUE); draw_text_align(ctx, ekLEFT, ekTOP); draw_text(ctx, ekFILL, "Text 文本 Κείμενο", 25, 25); code. img(text_single_line.png,0,.65).Single-line texts, with alignment and transformations. Drawing a text string several times. p.If the string to be displayed has new lines (character '\n') they will be taken into account and the text will be shown in several lines multext text_multiline. We can also obtain the measure in pixels of a block, useful to integrate the text with other primitives. code(cpp,multext).Dibujo de textos con saltos de línea. const char_t *text = "Text new line\n文字换行\nΓραμμή κειμένου"; real32_t width, height; draw_text(ctx, ekFILL, text, 25, 25); draw_text_extents(ctx, text, -1, &width, &height); code. img(text_multiline.png,0,.65).Texts with a '\n' character. Drawing several times of a text string containing new line characters. p.If the text does not contain new lines, it will be drawn continuously expanding horizontally. This may not be the most appropriate in long paragraphs, so we can set a maximum width, forcing its drawing in several lines blocktext text_paragraph. code(cpp,blocktext).Maximum width and internal alignment in text blocks. const char_t *text = "Lorem ipsum dolor sit amet...consequat"; draw_text_width(ctx, 200); draw_text_halign(ctx, ekLEFT); draw_text(ctx, ekFILL, text, 25, 25); draw_text_extents(ctx, text, 200, &width1, &height1); code. img(text_paragraph.png,0,.65).Text paragraphs with width limit. The maximum and real width obtained with draw_text_extents are shown. Drawing of text blocks with various widths. p.The text behaves like any other region: It is possible to combine the stroke drawing with the fill, or even use gradients using the parameter op from draw_text gradtext text_stroke_fill. code(cpp,gradtext).Text with dotted stroke and gradient fill. color_t c[2]; real32_t stop[2] = {0, 1}; real32_t dash[2] = {1, 1}; c[0] = kCOLOR_BLUE; c[1] = kCOLOR_RED; draw_line_dash(ctx, dash, 2); draw_line_color(ctx, kCOLOR_GREEN); draw_text_extents(ctx, "Gradient dashed text", -1, &width, &height); draw_fill_linear(ctx, c, stop, 2, 25, 0, 25 + width, 0); draw_text(ctx, ekFILLSK, "Gradient dashed text", 25, 250); code. img(text_stroke_fill.png,0,.65).Combining fill and stroke. Text with different border and fill styles. bq.The optimal way to draw texts is with solid color and ekFILL operation. This should be configured in the case that we have to make intensive use: browsers, word processors, reports, etc. Drawing contours or using gradients will have a performance penalty. h2.Drawing images p.Images generated procedurally or read from disk can be used as a drawing primitive more dimage draw_image.
