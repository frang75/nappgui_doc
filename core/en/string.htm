langrev. h1.Strings ep.Character strings in UTF-8, dynamically allocated. p.String objects contain UTF-8 characters strings dynamically allocated. Although it is possible to insert text strings directly into the source code or access them through the resource packages (respack_text), it is usual compose texts at runtime or store volatile data received through some input channel (keyboard, files, network, etc). The module strings.h offers support to work with this type of objects, as well as generic functions for the text manipulation. p.There are different ways of composing UTF8 strings, but the two most used constructors are str_c to create a copy of a C string and str_printf to compose a string from several fields, with the well-known format of the printf() function. code. String *str1 = str_c("This a static char array."); String *str2 = str_printf("Code: %s, Price %8.2f.", tc(product->code), product->price); str1 = "This a static char array." str2 = "Code: 456-34GH-JKL, Price 439.67." code. p.In the case that it is necessary to create more extensive texts from loops or other conditional, the most efficient way is to create a Stream and, later, obtain the String. code.
